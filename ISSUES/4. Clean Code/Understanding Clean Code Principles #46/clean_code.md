# Answer

**Simplicity** – Means writing code that solves a problem in the simplest possible way without unnecessary complexity. Simple code is easier to read, debug, and extend. Developers should avoid over-engineering and complicated solutions when a straightforward approach works.
**Readability** – Code is code that other developers can understand easily. It should use clear variable names, proper formatting, and logical structure so that someone unfamiliar with the system can still understand it quickly.
**Maintainability** – Code is easy to modify, fix, and extend in the future. Software projects evolve over time, so code should be organized in small, modular pieces to allow developers to update it without breaking the system.
**Consistency** – Means following the same coding style, naming conventions, and structure across the project. When the codebase has a consistent style, developers can read and understand it faster because everything follows predictable patterns.
**Efficiency** – Code performs tasks using optimal use of resources like time and memory. However, developers should avoid premature optimization. The goal is to write clean and readable code first, and then optimize when necessary.
